常见问题

1、 arm电脑安装哪个wine？
    有以下几种方案：
        a. box86+x86 wine，一般推荐这一方案
        b. 是exagear+wine x86（只支持鲲鹏）
        c. 运行arm wine，推荐用 wine ce 因为原版的 arm wine 只能运行 arm exe 但 wine ce 可以运行 x86 exe
        一般用户推荐使用方法1/2或者 wine ce，但 wine ce 需要系统的 libc6 比较新
2、麒麟系统能用吗
    能，但是openkylin要自己补依赖
3、字体问题怎么处理
wine 运行器的字体商店或者星火的安装 Windows 字体，在打包deb时推荐把字体打包在里面
4、deepin wine8 stable 能不能在arm用（box64）
    可以，但需要拉取box64 Github最新的代码编译
5、arm dwine6不能运行64位exe
    是的，不支持，同理也不支持64位wine容器。但可以安装64位wine+box64解决
6、Windows虚拟机配置工具点击安装无反应
    菜单栏里面找到安装日志然后到 https://gitee.com/gfdgd-xi/deep-wine-runner/issues 反馈
7、运行 wine提示libc6版本太老
    在菜单栏“运行库”里安装应用运行库，然后重启运行器选择使用运行器运行库运行xxx
8、我想要非X86架构 wine运行器离线包或者定制x86 wine运行器离线版
    因为平台特殊，需要单独联系开发者定制（一般是付费服务）
10、 我想装 .net framework 怎么办
    首先需要声明一点，.net framework 程序运行效果较差，不管是用 mono 还是微软原版的 .net framework
    可以安装mono（社区的开源实现），当然也可以安装微软原版的 .net framrwork
    均可在 Wine 运行器主界面的菜单栏“Wine”=》“在指定Wine、容器安装组件”处找到安装工具
    同理也有 Internet Explorer 的开源实现 gecko，但极不推荐安装微软原版的 Internet Explorer
